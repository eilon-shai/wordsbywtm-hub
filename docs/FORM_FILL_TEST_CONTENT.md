# Form-fill test content

Ready-to-paste sample content for a full manual run of every occasion — **1 organizer
(create form) + 3 contributors (share-link form)** per product. The text is realistic
and substantive (every memory clears the contributor guard: ≥ ~20 words and ≥ 2
sentences) so synthesis produces a genuine deliverable, not a refusal.

## How to use / gotchas
- **Emails must be unique per person.** One memory per email is enforced. The
  **organizer's email must differ from all contributor emails** — the public share
  link rejects the organizer's own address.
- Each contributor opens the **share link** (`/c/<shareToken>`) the organizer gets
  after creating the collection — they fill *Your name + Your email + relationship +
  memory* only.
- **Relationship** on the create form (and on the wedding/retirement/anniversary
  contributor forms) is a **dropdown** — pick the labelled option shown.
  Memorial's *contributor* relationship is free text.
- **Deadline** defaults to ~30 days out — leave it, or pick any date within a month.
- **Consent** checkbox is required on every form.
- For a real run use the **mock-payment** path (or sandbox); don't burn a live charge.
- All names/emails below are obviously fake on purpose, so stray rows are easy to spot
  and delete afterward.

## Quick abuse / edge-case checks (optional, while you're in here)
- **Organizer-own-email rejection:** try contributing on the share link using the
  organizer's email above — it should be refused.
- **One-per-email:** submit a second memory with an email already used — should be
  blocked/deduped.
- **Calm not-found:** open `/c/does-not-exist` and a deleted collection's link — both
  should show the warm "this link isn't active / has closed" screen.
- **Markdown-clean deliverable (venture-core 1.24.3):** the finished piece should be
  plain prose — no stray `#` title or `**bold**` on screen, in the PDF, the email, or
  the audio.
